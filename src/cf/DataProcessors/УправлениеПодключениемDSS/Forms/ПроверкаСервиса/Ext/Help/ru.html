Предназначена для проверки работоспособности сервиса DSS.

Проверка работоспособности позволяет проверить следующие операции:

  • Выбор сертификата - если в списке отсутствуют сертификаты, то необходимо инициировать выпуск нового сертификата, хранящийся в сервисе DSS.
  • Получить свойства сертификата – проверка получение свойства сертификата, хранящегося в сервисе.
  • Зашифровать произвольные данные – шифрование тестовых данных в формате CMS.
  • Зашифровать XML данные – шифрование тестовых данных в формате XML. Если операция закончилась неудачей, то сервис DSS не поддерживает операции в формате XML.
  • Расшифровать произвольные данные – расшифрование тестовых данных, которые были зашифрованы не предыдущем шаге.
  • Расшифровать XML данные – расшифрование тестовых данных, которые были зашифрованы не предыдущем шаге.
  • Подписать произвольные данные – формирование электронной подписи для тестовых данных в формате CMS.
  • Подписать XML данные – формирование электронной подписи для тестовых данных. Если операция закончилась неудачей, то сервис DSS не поддерживает операции в формате XML.
  • Проверить сертификат - проверяет выбранный на предыдущих шагах сертификат на валидность (срок действия, отзыв и прочее). Если операция завершилась неудачей, то сервис проверки не доступен или не настроен.
  • Проверить подпись – проверяет ранее сформированнную электронную подпись тестовых данных.
  • Проверить подпись XML - проверяет ранее сформированную электронную подпись в формате XML тестовых данных. Если операция завершилась неудачей, то сервис проверки не поддерживает операции проверки в формате XML.
  • Хешировать данные – вычисляет хеш-значение тестовых данных.
  • Подтвердить операцию – запускает процесс подтверждение произвольной операции. Порядок и содержание зависит от указанного имени шаблона подтверждения указанного в расширенных настройках сервера DSS.

В зависимости от настроек сервиса DSS для операций расшифрования и подписание может дополнительно запрашиваться подтверждение операции.
